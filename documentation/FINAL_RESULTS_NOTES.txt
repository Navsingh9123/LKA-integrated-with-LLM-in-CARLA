PROJECT OVERVIEW
----------------
This project compares four baseline path/lane tracking controllers against
AI/LLM-supervised versions of each, implemented in CARLA 0.9.15.

Baseline controllers:     Pure Pursuit, Stanley, PID, MPC
AI-supervised controllers: AI Pure Pursuit, AI Stanley, AI PID, AI MPC

The AI-supervised modes use a live Ollama LLM (llama3.2:1b, GPU-first) running
in a background daemon thread during the simulation. The LLM receives a
natural-language user request and returns bounded adjustments to high-level
controller parameters (e.g. lookahead distance, gain values, target speed).
The LLM never outputs a steering command directly — all steering is computed
by the deterministic controller from live vehicle state and route data.

IMPORTANT CAVEATS
--------------------------------------------------------------------------------

  [*] SPEED CONFOUND — AI Pure Pursuit and AI Stanley
      The LLM supervisor significantly reduced target_speed_kmh for these two
      controllers during their runs:
        AI Pure Pursuit: mean 9.63 km/h  vs baseline 12.12 km/h
        AI Stanley:      mean 7.98 km/h  vs baseline 12.11 km/h
      Lower speeds reduce lateral error because steering corrections have more
      time to act. Direct numerical comparison of lateral error for these pairs
      is confounded by speed and should be discussed carefully in the report.

  BASELINE SPEED
      All four baseline controllers targeted 15.0 km/h but achieved a mean of
      approximately 12.1 km/h. This is expected behaviour of the proportional
      throttle controller (Kp = 0.25, max_throttle = 0.45) in the road geometry
      of Town10HD_Opt.

  PID vs AI PID
      RMS lateral error is near-identical (0.0999 m both). The LLM produced
      conservative parameter changes that maintained safe performance without
      meaningful numerical improvement. This is a valid research finding.

  MPC vs AI MPC
      AI MPC shows a small increase in lateral error (0.1328 vs 0.1248 m RMS)
      but lower heading error RMS (1.890 vs 1.979 deg). The LLM traded a minor
      increase in cross-track offset for a smoother heading profile, consistent
      with the seeded user request.
